### ReactMediaRecorder/useReactMediaRecorder Options and Props API

Source: https://github.com/deltacircuit/react-media-recorder/blob/master/README.md

Detailed API documentation for the configuration options and props available for both the ReactMediaRecorder component and the useReactMediaRecorder hook, including types, defaults, and descriptions.

```APIDOC
audio:
  type: boolean | object (MediaTrackConstraints)
  default: true
  description: Can be either a boolean value or a MediaTrackConstraints object.

blobPropertyBag:
  type: object (BlobPropertyBag)
  default:
    if video is enabled: { type: "video/mp4" }
    if only audio is enabled: { type: "audio/wav" }
  description: An optional BlobPropertyBag dictionary specifying MIME type and line ending handling for the mediaBlob.

customMediaStream:
  type: MediaStream
  description: A media stream object itself (optional).

mediaRecorderOptions:
  type: object
  default: {}
  description: An optional options object passed to MediaRecorder. Has higher precedence than MIME type specified via audio or video props.

onStart:
  type: function()
  default: () => null
  description: A function invoked when the MediaRecorder starts.

onStop:
  type: function(blobUrl: string, blob: Blob)
  default: () => null
  description: A function invoked when the MediaRecorder stops, providing the blob and blob URL as parameters.

stopStreamsOnStop:
  type: boolean
  default: true
  description: Whether to stop all streams on stop.

render:
  type: function({ status: string, startRecording: function, stopRecording: function, mediaBlob: Blob }) => ReactElement
  default: () => null
  description: A function which accepts an object containing recording status, control functions, and media blob, returning a React element. (Only for ReactMediaRecorder component)

screen:
  type: boolean
  description: A boolean value. Lets you record your current screen. Screen capturing takes precedence over video capturing if both screen and video props are provided. Video prop (as MediaTrackConstraints) can be utilized by screen capture.

video:
  type: boolean | object (MediaTrackConstraints)
  default: false
  description: Can be either a boolean value or a MediaTrackConstraints object.

askPermissionOnMount:
  type: boolean
  default: false
  description: If set to true, will ask media permission on mounting.

Props available in the render function:
  error:
    type: string (enum)
    possible_values:
      - media_aborted
      - permission_denied
      - no_specified_media_found
      - media_in_use
      - invalid_media_constraints
      - no_constraints
      - recorder_error
    description: A string enum indicating various error conditions.
```

### ReactMediaRecorder API Reference

Source: https://github.com/deltacircuit/react-media-recorder/blob/master/README.md

Detailed reference for the properties and functions exposed by the `react-media-recorder` component, including status states, recording controls, media URLs, and live stream access.

```APIDOC
status: string enum
  Description: Current state of the media recorder.
  Possible values:
    - media_aborted
    - permission_denied
    - no_specified_media_found
    - media_in_use
    - invalid_media_constraints
    - no_constraints
    - recorder_error
    - idle
    - acquiring_media
    - recording
    - stopping
    - stopped

startRecording: function
  Description: Initiates the media recording process.

pauseRecording: function
  Description: Pauses the ongoing media recording.

resumeRecording: function
  Description: Resumes a paused media recording.

stopRecording: function
  Description: Halts the media recording and finalizes the media blob.

muteAudio: function
  Description: Mutes the audio tracks of the recording.

unmuteAudio: function
  Description: Unmutes the audio tracks of the recording.

mediaBlobUrl: string (blob URL)
  Description: A URL pointing to the recorded media blob, suitable for <audio />, <video />, or <a /> elements.

clearBlobUrl: function
  Description: Clears any existing generated blob URL and resets the recorder to its initial 'idle' state.

isMuted: boolean
  Description: Indicates whether the audio is currently muted.

previewStream: MediaStream
  Description: A muted MediaStream for live video preview, attachable to a <video /> element. Muted by design to prevent audio feedback.

previewAudioStream: MediaStream
  Description: A live audio stream accessible for use with Web Audio API features like AudioContext and AnalyzerNode for sound visualizations.
```

### Record Media with ReactMediaRecorder Render Prop

Source: https://github.com/deltacircuit/react-media-recorder/blob/master/README.md

Demonstrates how to use the ReactMediaRecorder component with a render prop to control recording status, start/stop actions, and display the recorded media.

```javascript
import { ReactMediaRecorder } from "react-media-recorder";

const RecordView = () => (
  <div>
    <ReactMediaRecorder
      video
      render={({ status, startRecording, stopRecording, mediaBlobUrl }) => (
        <div>
          <p>{status}</p>
          <button onClick={startRecording}>Start Recording</button>
          <button onClick={stopRecording}>Stop Recording</button>
          <video src={mediaBlobUrl} controls autoPlay loop />
        </div>
      )}
    />
  </div>
);
```

### Displaying Live Video Preview with ReactMediaRecorder

Source: https://github.com/deltacircuit/react-media-recorder/blob/master/README.md

This snippet demonstrates how to use the `previewStream` prop from `react-media-recorder` to display a live video feed in a `<video>` element. It includes a `VideoPreview` component to handle stream assignment and an `App` component showing its integration within the `ReactMediaRecorder` render prop.

```tsx
const VideoPreview = ({ stream }: { stream: MediaStream | null }) => {
  const videoRef = useRef<HTMLVideoElement>(null);

  useEffect(() => {
    if (videoRef.current && stream) {
      videoRef.current.srcObject = stream;
    }
  }, [stream]);
  if (!stream) {
    return null;
  }
  return <video ref={videoRef} width={500} height={500} autoPlay controls />;
};

const App = () => (
  <ReactMediaRecorder
    video
    render={({ previewStream }) => {
      return <VideoPreview stream={previewStream} />;
    }}
  />
);
```

### Record Media with useReactMediaRecorder Hook

Source: https://github.com/deltacircuit/react-media-recorder/blob/master/README.md

Illustrates how to use the useReactMediaRecorder hook to manage recording state, actions, and media display in a functional React component.

```javascript
import { useReactMediaRecorder } from "react-media-recorder";

const RecordView = () => {
  const { status, startRecording, stopRecording, mediaBlobUrl } =
    useReactMediaRecorder({ video: true });

  return (
    <div>
      <p>{status}</p>
      <button onClick={startRecording}>Start Recording</button>
      <button onClick={stopRecording}>Stop Recording</button>
      <video src={mediaBlobUrl} controls autoPlay loop />
    </div>
  );
};
```
